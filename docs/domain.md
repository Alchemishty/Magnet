# Domain Glossary

This document defines the business domain of Magnet — an agentic platform for mobile game UA creative production.

---

## Core Entities

### User

A registered person who accesses the platform. Has an email, name, organization, and role. Authenticated via Clerk.

### Project

A workspace for a single game's UA creative production. Belongs to a User. Contains a GameProfile, uploaded Assets, generated CreativeBriefs, and RenderJobs.

### GameProfile

The identity card of a mobile game. Captures genre, target audience, core mechanics, art style, brand guidelines, competitors, and key selling points. This is the primary input to the Concept Agent — the richer the profile, the better the creative output.

### Asset

An uploaded file associated with a Project. Types: gameplay footage (video), screenshots, logos, character art, audio. Stored in S3/MinIO. Assets are the raw material for the COMPOSE strategy.

### CreativeBrief

The blueprint for a single video ad. Contains: hook type, narrative angle, script, voiceover text, CTA options, target emotion, target format/duration, and a scene-by-scene plan with strategy tags (COMPOSE/GENERATE/RENDER). Can be generated by the Concept Agent or created manually. Lifecycle: draft → approved → producing → complete.

### RenderJob

Tracks the production of one video ad from a CreativeBrief. Contains: status (queued/rendering/done/failed), template reference, Composition JSON, output S3 key, render duration, and error message if failed. The Video Agent creates and manages RenderJobs.

### Composition JSON

The machine-readable contract between AI planning and video rendering. A layered timeline describing every element in the final video: clips, text overlays, audio tracks, transitions, effects. Stored as JSONB in the RenderJob record.

## Agentic Pipeline Concepts

### Concept Agent

The AI pipeline that generates creative briefs. Three chained steps:

1. **STRATEGIZE** — analyze game profile, identify hook categories, select emotional angles
2. **EXPAND** — generate full briefs with scene plans for each direction
3. **DIVERSIFY** — review all briefs, eliminate redundancy, suggest mutations

### Video Agent

The AI pipeline that produces video ads from approved briefs. Four phases:

1. **PLAN** — parse scene plan, match assets, build execution dependency graph
2. **PREPARE** — execute all scene preparation tasks (parallel where possible)
3. **ASSEMBLE** — build FFmpeg filter graph, layer clips, mix audio, encode
4. **POST-PROCESS** — encode to ad specs, generate thumbnail, upload to S3

### Three Scene Strategies

**COMPOSE** — Use real uploaded assets. Trim gameplay footage, crop screenshots, overlay text. Fast, authentic, essentially free after infrastructure.

**GENERATE** — AI-create elements that don't exist. Voiceovers (TTS), music, AI avatars, novel imagery. This is where novelty and scale come from. Uses external provider APIs.

**RENDER** — Programmatically code-render scenes. Animated text sequences, UI mockups, fake gameplay mechanics, progress bars. Too structured for generative models, too dynamic for static templates.

## Hook Taxonomy

Categorized ad opening strategies built into the Concept Agent's system prompts:

| Hook Category | Examples | Best for |
|---------------|----------|----------|
| Fail/Challenge | "I can't get past level 5", "99% fail" | Casual, puzzle |
| Satisfaction | Oddly satisfying loops, ASMR-style | Idle, merge, casual |
| Comparison | "Level 1 vs Level 100", "Noob vs Pro" | RPG, strategy |
| Emotional | Story-driven narrative, character journey | RPG, narrative |
| UGC-style | Fake reaction, "I found this game and..." | All genres |
| Fake gameplay | Pin-pull, choice-based, exaggerated | Casual, hyper-casual |
| FOMO/Social | "Everyone's playing this", trending | All genres |
| Tutorial bait | "Here's a trick most players don't know" | Strategy, mid-core |

This taxonomy is extensible.

## Business Rules

- A Project must have a GameProfile before concept generation can be triggered.
- A CreativeBrief must be in "approved" status before video production can start.
- A RenderJob that fails is marked "failed" with an error message — never silently degraded.
- Assets are immutable once uploaded. New versions create new Asset records.
- Composition JSON schema changes require a decision record.
- Target format for MVP: 9:16 vertical, 15-30 seconds.
- External API calls (LLM, TTS, music) use automatic retry with exponential backoff.

---

## User Roles

For MVP, single-user per project. Multi-user collaboration is deferred to Phase 3.

- **Creator** — full access: create projects, upload assets, generate concepts, produce videos, download creatives
